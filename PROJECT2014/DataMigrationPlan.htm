Data Migration Plan

1.       Pre Planning

 

a.       The purpose of the Data Migration Plan is to migrate all the data from the SQL Server to the Oracle database. The data that we migrate is real time data of Dillards. This data helps to understand the questions that most executives have.

b.      The data consists of Dillards sales information from 2004 to 2005. The data include Universal Product Code (UPC), Stock Keeping Unit (UPC), Department (dept), vendor, brand etc.

c.       Data migration is carried out every year to move previous years data to historic DB.

d.      Allocation of human resources for data migration.

e.       The technical resources involved in the project:

 

         Remote Desktop Connection

         Server Management Studio

         Oracle SQL Developer

         CISCO VPN Connection for secure connection to UIS

         Filezilla

         Microsoft Excel

         Aptana

         Notepad

         Microsoft Word

 

2.       Analysis & Design

 

a.       Steps followed in data migration

         Login to the Remote Desktop by providing Gateway Server information and credentials

         Login to SQL Server Management Studio by providing Database Server information and credentials

         Open UA_Dillards Databases folder.

         Create a New query to extract required data.

         Click Execute to execute the query.

         When results show up, right click anywhere on the result page and select Save Results as to save the result in a .csv file

         Save the file on the remote desktop.

         Then move the file to your personal system.

         Verify the data

         Transform the data to eliminate any inconsistencies

         Create required tables on UIS Oracle Server using SQL Developer

         Load the data on to UIS Oracle Server

         Validate the inserted data

 

b.      Validate the data

         Check that tables have been created

         Check that all previous tables are accounted for and functioning correctly

         Run sample queries to test database functionality and availability

 

c.       Mapping

         Compare new table columns to previous years, to ensure consistency.

         Create table relationships per original design.

         Run sample queries to test relationship functionality

         Update any existing queries or files that reference historical player data so that the most recent year is included.

         Run sample queries to test changes to above referenced queries and files.

 

3.       Post Migration Activities

 

a.       Check for table row count

b.      Check for Database Integrity

c.       Check for referential integrity

d.      Check if the data is loaded correctly

e.       Check for constraints

f.       Check functionality by running some queries

CREATE TABLE STATEMENTS

CREATE TABLE SKUINFO (SKU Int NOT NULL,

DEPT Int NOT NULL,

VENDOR Char(7) NULL,

BRAND Char(9) NULL,

UPC Char(15) NULL,

DEPTDESC Char(8) NULL,

CONSTRAINT SKUINFOPK PRIMARY KEY(SKU));

 

CREATE TABLE STORE_MSA (Store Int NOT NULL,

City Char(20) NULL,

State Char(2) NULL,

Zip Char(5) NULL,

MSA Int NULL,

MSA_NAME Char(40) NULL,

CONSTRAINT STORE_MSAPK PRIMARY KEY(Store));

 

CREATE TABLE TRNSACT (SEQ CHAR(9) NOT NULL,

REGISTER INT NOT NULL,

TRANNUM CHAR(5) NOT NULL,

SALEDATE TIMESTAMP NOT NULL,

QUANTITY INT NOT NULL,

SPRICE NUMBER(6,2) NOT NULL,

AMT NUMBER (7,2) NOT NULL,

STORE INT NOT NULL,

SKU INT NOT NULL,
CONSTRAINT TRNSACT_PK PRIMARY KEY(SEQ,REGISTER,TRANNUM,SALEDATE),

CONSTRAINT TRNSACT_FK1 FOREIGN KEY(Store)

REFERENCES STORE_MSA(Store),

CONSTRAINT TRNSACT_FK2 FOREIGN KEY(SKU)

REFERENCES SKUINFO(SKU));
